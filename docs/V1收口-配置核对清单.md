## 一、使用说明

- 本清单对应任务书 `T1.5`，用于上线前统一核对支付、小程序、订阅消息、企业微信等外部配置。
- 本清单只覆盖婚庆 `V1` 主链路收口必需项，不展开 `V1.5 / V2` 增量能力。
- 任一 `D1-D3` 未完成时，仅允许继续整理台账、补录配置、准备联调数据，不启动站外通知验收。

## 三、站外通知前置判定

### 1. 买家订阅消息联调前置

- `C2` 已完成。
- `C3` 已完成。
- 至少一个买家测试账号已完成小程序登录并可拉起订阅授权。
- `消息通知配置` 中对应场景的订阅消息通道已开启。

### 2. 服务人员企业微信联调前置

- `C4` 已完成。
- `C5` 已完成。
- 至少一个服务人员账号已绑定真实 `user_id`，并在服务人员主档中配置 `work_wechat_userid`。
- 企业微信应用具有向对应成员发消息的权限。

## 四、联调核对动作

### 1. 微信支付

- [ ] 在管理后台确认婚庆订单使用的支付方式可见。
- [ ] 使用婚庆订单详情页触发在线支付，确认能拉起支付流程。
- [ ] 支付完成后确认订单状态进入待履约。
- [ ] 回后台工作台、订单管理、通知日志确认数据已同步更新。

### 2. 小程序订阅消息

- [ ] 在下单前拉起买家生命周期场景授权：`201、202、203、204、205、207、208、211`。
- [ ] 在改期提交前拉起结果场景授权：`210`。
- [ ] 在退款提交前拉起结果场景授权：`214、215`。
- [ ] 在评价提交前拉起审核结果场景授权：`213`。
- [ ] 对照日志确认模板 ID、字段映射、跳转页与订单字段一致。

### 3. 企业微信应用消息

- [ ] 对照服务人员档案确认 `work_wechat_userid` 已配置。
- [ ] 触发 `201、205、206、209、212` 五个关键待办节点。
- [ ] 在通知日志确认请求报文、返回报文、错误原因均可追踪。
