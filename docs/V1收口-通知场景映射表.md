## 一、说明

- 本表按 `ServiceOrderService` 实际触发点整理，不以 `la_notice_setting.recipient` 单独作为联调范围判断。
- 买家订阅消息使用 `/wedding/notice/subscribeTemplates` 获取模板，并在用户动作前调用 `requestWeddingSubscribeMessages()` 拉起授权。
- 服务人员企业微信消息由服务端自动发送，不依赖前端授权。

## 四、买家订阅授权策略

### 1. 已调整的授权入口

- 订单预览页：统一请求 `201、202、203、204、205、207、208、211`。
- 订单详情页改期提交前：请求 `210`。
- 订单详情页退款提交前：请求 `214、215`。
- 评价提交页：请求 `213`。

### 2. 策略说明

- 买家端只在用户主动操作前拉起授权，不新增独立的消息演示页。
- 后续服务端事件通知是否实际送达，取决于：
  - 通知场景是否启用；
  - 模板 ID 与字段映射是否已配置；
  - 用户是否接受该模板授权；
  - 微信接口返回结果是否成功。

## 五、联调核对点

- [ ] 场景触发后主流程不因消息失败中断。
- [ ] 日志表中 `request_data`、`response_data`、`error_message` 可用于排查。
- [ ] 买家消息跳转页均回到订单详情页。
- [ ] 服务人员消息跳转地址与待办场景一致。
- [ ] 模板字段替换结果与订单真实字段一致。
